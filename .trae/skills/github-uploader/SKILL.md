# GitHub 代码上传器

## 技能功能

本技能提供自动化流程帮助用户将本地代码上传到 GitHub：
1. 检查并配置 GitHub CLI 认证
2. 交互式创建仓库（名称、描述、可见性）
3. 自动推送代码到 GitHub

适用于快速将本地项目发布到 GitHub。

## ⚙️ 核心命令实现

### 环境检查

```bash
# 检查 gh 是否安装
which gh

# 检查版本
gh --version

# 检查登录状态
gh auth status
```

### 认证流程

```bash
# 浏览器登录（推荐）
gh auth login

# 使用 Token 登录
echo "YOUR_TOKEN" | gh auth login --with-token

# 配置 git 使用 gh 认证
gh auth setup-git
```

### 创建仓库

```bash
# 创建公开仓库
gh repo create <name> --public --source=. --push

# 创建私有仓库
gh repo create <name> --private --source=. --push

# 仅创建仓库，不推送
gh repo create <name> --public --description="描述"

# 交互式创建（会提示输入）
gh repo create
```

### 推送代码

```bash
# 如果仓库已存在，添加 remote 并推送
git remote add origin https://github.com/username/repo.git
git branch -M main
git push -u origin main
```

## 📋 上传内容控制

### 当前项目配置

根据 `.gitignore` 配置，当前项目只上传：
- `.trae/skills/image-prompt-generator/` 目录
- `README.md`

### 自定义上传内容

如需修改上传内容，编辑 `.gitignore` 文件：

```gitignore
# 保留特定技能
.trae/skills/
!.trae/skills/your-skill/

# 排除敏感文件
*.env
*.key
secrets/
```

## ⚠️ 注意事项

1. **Token 安全**：
   - 不要将 Token 提交到代码库
   - 使用浏览器登录更安全
   - Token 存储在系统密钥链中

2. **仓库名称**：
   - 只能包含小写字母、数字、连字符
   - 不能与现有仓库重名
   - 长度不超过 100 字符

3. **可见性选择**：
   - Public：任何人都可以看到，适合开源项目
   - Private：只有你和协作者可以看到

4. **Git 配置**：
   - 首次使用需配置用户名和邮箱
   - gh 会自动处理 git 配置

## 🔧 故障排除

### 问题：gh 未安装

**解决方案：**
```bash
# Windows (winget)
winget install GitHub.cli

# macOS
brew install gh

# Linux
sudo apt install gh
```

### 问题：认证失败

**解决方案：**
```bash
# 清除旧认证
gh auth logout

# 重新登录
gh auth login
```

### 问题：推送被拒绝

**解决方案：**
```bash
# 检查远程仓库是否存在
gh repo view

# 如果是权限问题，检查 token 权限
gh auth status

# 重新配置 git
gh auth setup-git
```

### 问题：仓库已存在

**解决方案：**
- 更换仓库名称
- 或删除 GitHub 上的旧仓库后重试
